# 🔄 업데이트 배포 방법

## 준비 (최초 1회)

1. GitHub 계정 만들기: https://github.com
2. 새 레포지토리 만들기: `byultddaogi-nametag` (Public으로 설정)
3. 이 폴더의 파일들을 레포에 올리기:
   - `main.py`
   - `version.json`
   - `README.md`
   - `.github/workflows/build.yml`

4. `main.py` 상단의 두 줄 수정:
   ```python
   GITHUB_USER  = "실제_깃허브_아이디"
   GITHUB_REPO  = "byultddaogi-nametag"
   ```

## 업데이트 배포 방법 (매번)

### Step 1: `version.json` 수정

```json
{
  "version": "1.1.0",
  "changelog": "## v1.1.0\n\n- 새로운 기능 추가\n- 버그 수정"
}
```

> changelog는 마크다운 형식으로 작성하면 앱에서 예쁘게 표시됩니다.

### Step 2: `main.py` 상단 버전 수정

```python
APP_VERSION = "1.1.0"
```

### Step 3: GitHub에 push + 태그 달기

```bash
git add .
git commit -m "v1.1.0 업데이트"
git tag v1.1.0
git push origin main --tags
```

### Step 4: 자동 빌드 기다리기

- GitHub Actions가 자동으로 Windows exe 빌드
- 빌드 완료 후 Releases에 자동 업로드 (약 5~10분)
- 이후 앱 실행 시 사용자에게 업데이트 알림 자동 표시
